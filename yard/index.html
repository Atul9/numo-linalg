Numo::Linalg : Linear Algebra library with BLAS/LAPACK binding to Numo::NArray

GitHub

Under development!

Introduction

This is a binding of LAPACK with Numo::NArray .

Numo::Linalg API

  • Matrix and vector products

    • dot, matmul

  • Decomposition

    • cholesky, qr, svd

  • Matrix eigenvalues

    • eig, eigh, eigvals, eigvalsh

  • Norms and other numbers

    • norm, det, matrix_rank, slogdet

  • Solving equations and inverting matrices

    • solve, lstsq, inv

More functions to come

Low-level modules

  • Numo::Linalg::Blas - Low-level BLAS functions

  • Numo::Linalg::Lapack - Low-level LAPACK functions

Installation

  • Install Numo::NArray

  • Install LAPACK or compatible packages.

    • Numo::Linalg requires C-interface CBLAS and LAPACKE interface. These are included in LAPACK package.

    • It is recommended to use one of following faster libraries:

      • ATLAS

      • OpenBLAS

      • Intel MKL

  • Install Numo::Linalg

$ git clone https://github.com/ruby-numo/linalg.git
$ cd linalg
$ rake build
$ gem install pkg/numo-linalg-*.gem

Loading backend library

  • Numo::Linalg opens dynamic libraries of BLAS and LAPACK at runtime. This design allows you to change backend libraries without re-compiling Numo::Linalg.

  • Note that the performance depends on the backend libraries as shown in benchmark.

  • Dynamic loading example:

require "numo/linalg"
require "fiddle"
Fiddle.dlopen("libblas.so")  # referenced from CBLAS
Fiddle.dlopen("liblapack.so")  # referenced from LAPACKE
Numo::Linalg::Blas.dlopen("libcblas.so")  # contains cblas_* function
Numo::Linalg::Lapack.dlopen("liblapacke.so")  # containes LAPACKE_* function

This default behavior is defined in “numo/linalg/use/lapack.rb”

require "numo/linalg/use/lapack"

  • Load Atlas:

require "numo/linalg/use/satlas"

  • Load OpenBLAS:

require "numo/linalg/use/openblas"
